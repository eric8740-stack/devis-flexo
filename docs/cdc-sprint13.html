— 02 / Règle centrale

La règle métier : silhouette

Toute la logique de réutilisation des plaques repose sur une seule règle métier centrale, héritée du savoir-faire ICE. Cette règle distingue deux types de plaques avec des conséquences différentes.

Définition d'une plaque "silhouette"

Une plaque est silhouette si au moins une des conditions suivantes est vraie :

  • Forme autre que rectangle (rond, ovale, étoile, profilée, etc.)
  • Rayon des angles ≠ 2 mm et ≠ 3 mm (seuls 2 et 3 mm sont standards)
  • Présence de prédécoupe entre étiquettes
  • Présence de micro-perforation entre étiquettes
  • Pose mixte (plusieurs étiquettes différentes sur la même plaque)

Conséquence métier

— Règle d'or

Une plaque silhouette est rattachée à un client unique (champ client_proprietaire_id). Elle n'est jamais proposée à un autre client par le moteur de recherche, même si le format géométrique matche. Une plaque standard (rectangle, rayon 2 ou 3, sans perf, mono-pose) est en revanche partagée entre tous les clients de l'imprimerie.

Pourquoi cette règle

  • Les distances de prédécoupe et micro-perforation varient d'un client à l'autre (machines de pose différentes)
  • Le rayon est une signature visuelle de l'étiquette du client, non modifiable sans son accord
  • Les formes non-rectangulaires sont systématiquement signature client (logo de marque)
  • Une pose mixte par définition contient les références d'un client donné

Le critère matière reste obligatoire dans les deux cas

Pour qu'une plaque (standard ou spéciale) soit réutilisable, la matière demandée doit correspondre à la matiere_compatible enregistrée. Le filet entre-fer 0,8 mm est universel chez les fabricants de plaques, mais le profil de découpe doit être adapté au substrat (papier couché ≠ BOPP ≠ thermique).

— 03 / Saisie devis

Saisie d'une demande de devis

Le formulaire de saisie est volontairement segmenté en 4 étapes pour éviter la surcharge cognitive. Tous les champs ont une valeur par défaut paramétrée par l'imprimerie, surchargeable par le commercial.

Étape 1 — Étiquette

  • Client (lien table clients) + référence interne
  • Forme : rectangle / rectangle arrondi / rond / ovale / spéciale
  • Format : laize (mm) × hauteur (mm)
  • Rayon des angles (mm) — si forme rect arrondi ou spéciale
  • Quantité totale (étiquettes)
  • Tolérance quantité (% — standard flexo ±5 %)

Étape 2 — Support & adhésif

  • Matière (liste déroulante depuis catalogue matières)
  • Grammage / épaisseur
  • Adhésif (permanent / repositionnable / renforcé / enlevable / contact alimentaire / surgelé)
  • Certifications sanitaires & réglementaires (FDA, EU 10/2011, BPF pharma)
  • Certification environnementale demandée par le client : aucune / FSC / PEFC / les deux
— Logique FSC / PEFC

Pour qu'une étiquette puisse porter le logo FSC ou PEFC, deux conditions doivent être réunies simultanément : (1) la matière sélectionnée dans le catalogue doit être certifiée pour cette norme, ET (2) l'imprimerie elle-même doit avoir une certification de chaîne de contrôle FSC ou PEFC active (déclarée dans ses paramètres entreprise). Si l'une des deux conditions n'est pas remplie, FlexoCompare affiche une alerte au commercial avant validation du devis.

Étape 3 — Mise en forme bobine

  • Intervalle laize (mm) — default entreprise, modifiable. Standard 3 ou 4 mm.
  • Intervalle développé (mm) — default entreprise, modifiable.
  • Diamètre mandrin (mm) : 25 / 38 / 40 / 76 / 100
  • Diamètre extérieur max bobine (mm) : 127 / 200 / 250 / 300
  • Nombre étiquettes par bobine (saisi ou calcul auto selon Dmax)
  • Sens d'enroulement : intérieur / extérieur
  • Sens de lecture : 1 / 2 / 3 / 4 (norme imprimerie)

Étape 4 — Impression & finitions

  • Nombre de couleurs : 1 à 8 + Pantone(s) spot + blanc + vernis
  • Type d'encre : UV / eau / solvant / faible migration alimentaire
  • Vernis : aucun / total mat / total brillant / sélectif
  • Prédécoupe entre étiquettes : oui / non V1
  • Micro-perforation : oui / non
  • Pose mixte : oui / non + liste références si oui
  • BAT physique requis : oui / non
  • Délai souhaité : standard / express / date impérative
  • Fréquence : unique / récurrente

Calcul automatique de est_silhouette

est_silhouette =
  forme != "rectangle"
  OU rayon_angles NOT IN (2, 3)
  OU has_predecoupe == TRUE
  OU has_microperforation == TRUE

Ce calcul s'exécute en temps réel à la saisie pour orienter le moteur de recherche en aval.

— 03b / Optimisation de pose · CŒUR DU SYSTÈME

Moteur d'optimisation de pose machine

Avant toute recherche de plaque et avant tout calcul de prix, FlexoCompare exécute le moteur d'optimisation de pose. C'est le cœur du système. Il regarde ce que le parc machine de l'imprimerie sait faire de mieux avec le format demandé, et propose au commercial les meilleures configurations de production.

— Pourquoi c'est central

Un même format d'étiquette peut être produit de 5 manières différentes selon les cylindres magnétiques et les machines disponibles. Chaque manière a un coût matière et un coût machine différents. Le moteur trouve la configuration la plus rentable, et propose au commercial des alternatives de format (négociation client) qui pourraient économiser davantage.

Concept de base — la "pose"

Une pose est le nombre d'étiquettes répétées sur un tour de cylindre.

  • Pose développé = nombre d'étiquettes alignées dans le sens de la rotation (autour du cylindre)
  • Pose laize = nombre d'étiquettes alignées dans le sens transverse (largeur de la bobine)
  • Pose totale = pose développé × pose laize = nombre d'étiquettes produites par tour de cylindre

Inputs du moteur

  • Format demandé : largeur × hauteur étiquette
  • Intervalle laize mini souhaité (default 3 mm)
  • Intervalle développé mini souhaité (default 3 mm)
  • Matière (pour calculer la gâche matière)
  • Quantité commandée
  • Parc machine de l'imprimerie : cylindres magnétiques disponibles (développés) + machines avec leurs laizes utiles et vitesses
  • Table qualité d'échenillage de l'imprimerie : courbe de scoring intervalle dev → coefficients vitesse/gâche

Concept critique — qualité d'échenillage

L'échenillage est le squelette de papier qui reste entre les étiquettes après découpe. Quand l'intervalle développé entre deux étiquettes est trop faible, le squelette est fragile et casse à grande vitesse — il faut ralentir la machine pour éviter les casses, ce qui coûte du temps machine et augmente la gâche matière (chaque casse = redémarrage avec déchet).

Inversement, quand l'intervalle est trop grand, la matière est gâchée car non utilisée pour des étiquettes.

⚖️ Règle de compensation laize / développé

— Mécanique de l'échenillage

Quand l'intervalle développé est forcé d'être grand (cylindre non idéal), le squelette de papier devient fragile dans le sens du déroulé. Pour consolider l'échenillage, on peut élargir l'intervalle laize (entre colonnes d'étiquettes) afin d'avoir plus de matière transverse qui tient l'ensemble.

Cette règle de compensation permet de récupérer une partie de la pénalité de vitesse causée par un grand intervalle dev — au prix éventuel de perdre une pose laize. Le moteur fait le calcul du trade-off automatiquement.

🍌 Filtre amont — effet banane (largeur de plaque)

— Règle métier issue de l'expertise ICE

Plus la plaque est large (laize), plus elle nécessite un cylindre magnétique avec un développé minimum pour ne pas se déformer en arc sur le cylindre. C'est un effet mécanique : une plaque large sur un petit cylindre se courbe en banane sous l'effet de la courbure et de la pression, ce qui dégrade la qualité d'impression et de découpe.

L'effet banane est un filtre dur : un cylindre exclu est définitivement exclu. Pas de compromis possible.

Notez que la courbe n'est pas linéaire : il y a un saut entre les paliers 250-300 (Z=120) et 300-350 (Z=160). Ce saut reflète un seuil physique de rigidité du cylindre — il ne peut pas être extrapolé par formule, c'est une donnée empirique métier.

Position du filtre dans l'algorithme

L'effet banane est appliqué en tout premier, avant même les calculs d'échenillage, parce qu'il élimine définitivement certains cylindres. Inutile de calculer un score pour un cylindre qui sera de toute façon exclu.

Algorithme principal

FONCTION optimiser_pose(format, intervalle_min, matiere, qte, parc_machine, 
                       table_echenillage, table_compensation, table_effet_banane) :

  resultats = []

  POUR chaque cyl DANS parc_machine.cylindres_actifs :

    nb_poses_dev = floor(cyl.developpe_mm / (format.hauteur + intervalle_min))
    SI nb_poses_dev == 0 : CONTINUE
    
    intervalle_dev_reel = (cyl.developpe_mm / nb_poses_dev) - format.hauteur
    SI intervalle_dev_reel < intervalle_min :
      nb_poses_dev -= 1
      SI nb_poses_dev == 0 : CONTINUE
      intervalle_dev_reel = (cyl.developpe_mm / nb_poses_dev) - format.hauteur

    qualite_base = table_echenillage.lookup(intervalle_dev_reel)
    intervalle_laize_souhaitable = table_compensation.lookup(intervalle_dev_reel)

    POUR chaque mach DANS parc_machine.machines_compatibles(cyl) :
      
      nb_poses_laize_max = floor((mach.laize_utile + intervalle_min) 
                                 / (format.largeur + intervalle_min))
      SI nb_poses_laize_max == 0 : CONTINUE
      
      POUR variante DANS [nb_poses_laize_max, nb_poses_laize_max - 1, nb_poses_laize_max - 2] :
        SI variante <= 0 : CONTINUE
        
        // ⚠️ NOUVEAU — Calcul de la largeur de plaque effective
        largeur_plaque_mm = variante * format.largeur 
                          + (variante - 1) * intervalle_laize_reel
        
        // ⚠️ NOUVEAU — Filtre EFFET BANANE
        z_mini_requis = table_effet_banane.lookup(largeur_plaque_mm)
        SI cyl.developpe_mm < z_mini_requis :
          CONTINUE  // Cylindre exclu : plaque trop large pour ce développé
        
        intervalle_laize_reel = (mach.laize_utile - variante * format.largeur) 
                                / (variante - 1) SI variante > 1 SINON 0

        // ... reste du calcul (échenillage, consolidation, score) inchangé
        
        resultats.append({
          ...,
          largeur_plaque_mm: largeur_plaque_mm,
          z_mini_effet_banane: z_mini_requis,
          ...
        })

  // ⚠️ NOUVEAU — Top 3, mais SANS triche : si moins de 3 candidats, on affiche ce qu'on a
  top_candidats = sorted(resultats, par score DESC)[:3]
  
  nb_candidats_restants = length(top_candidats)
  message_filtrage = SI nb_candidats_restants < 3 :
                       "Effet banane et contraintes parc machine ont éliminé certaines options. 
                        Seulement " + nb_candidats_restants + " configuration(s) viable(s)."
                     SINON : null

  RETOURNER {
    configurations_optimales: top_candidats,
    nb_candidats: nb_candidats_restants,
    message_filtrage: message_filtrage,
    propositions_negociation_client: ...,
    alertes_investissement: ...,
    alerte_qualite_production: ...
  }

— Règle d'affichage : pas de triche

Le moteur propose toujours jusqu'à 3 configurations. Si l'effet banane et les autres filtres laissent moins de 3 candidats viables, on affiche 2, ou 1, ou même 0 — mais on ne dégrade JAMAIS les contraintes pour faire du remplissage. 📏 Règle métier — intervalle laize max ~5 mm

— Limite pratique standard

En pratique standard flexo, l'intervalle laize entre poses est limité à 5 mm maximum. Au-delà, ce n'est plus un intervalle utile, c'est de la matière perdue inutilement. Si on a beaucoup d'espace libre côté laize, on préfère :

  • Soit ajouter une pose laize supplémentaire (si format compatible)
  • Soit accepter de la chute latérale sur les bords de la bobine (rognage en sortie)
  • Soit changer de cylindre ou suggérer un autre format au client

Exception : demande client spéciale (forme complexe, livret avec marges spécifiques, marquage en bord). Dans ce cas, intervalle laize hors barème = silhouette.

Concept de chute latérale (bord de bobine)

Quand la laize utile n'est pas un multiple parfait de (format + intervalle), il reste de la matière non utilisée sur les bords extérieurs de la bobine. On appelle ça la chute latérale. Elle est :

  • Distincte de l'intervalle entre poses (qui est entre étiquettes)
  • Rognée en sortie de machine
  • Comptabilisée comme gâche matière
  • Paramétrable : marge mini bord (default 3 mm chaque côté)

Barème de compensation mis à jour (avec plafond 5 mm)

Puisque l'intervalle laize max est ~5 mm en pratique, le bonus de consolidation s'applique selon une règle simplifiée :

Intervalle devIntervalle laize requis pour bonusBonus appliqué
≤ 4 mm≥ 3 mmAucun (déjà optimal)
5 mm≥ 4 mmCoef vitesse +15 %
6–8 mm≥ 5 mm (max)Coef vitesse +20 %
> 8 mm≥ 5 mm (max)Coef vitesse +25 % (consolidation max possible)

Au-delà de 5 mm de laize, plus de bonus supplémentaire — c'est la limite physique. Si l'intervalle dev est très grand, on accepte simplement la pénalité résiduelle.

Exemple concret corrigé du trade-off

— Cas type : étiquette 70×70, qté 30 000, cyl 95 mm (intervalle dev = 15 mm forcé)

Machine laize utile 226 mm :

  • Variante 1 — 3 poses laize · 3×70 = 210 mm + 2×5 mm intervalles + 6 mm bords = 226 mm pile. Intervalle laize 5 mm (max) → consolidation atteinte, coef vitesse 0,72 · score 38
  • Variante 2 — 2 poses laize · 2×70 = 140 mm + 5 mm intervalle + 81 mm de chute latérale. Coef vitesse 0,75 mais énorme gâche matière → score 22

→ Variante 1 RETENUE : meilleure utilisation matière, consolidation à 5 mm (max possible), production optimale dans les contraintes.

🚗 Confort de roulage — rayon et disposition des poses

— Métaphore de la roue qui monte un trottoir

Une roue qui aborde un trottoir face → choc frontal → mal. Une roue qui aborde une rampe progressive → roulage doux → confortable.

L'outil de découpe en rotation subit exactement la même physique : plus les angles d'attaque sont vifs, plus chaque tour génère un choc qui force à ralentir la machine, use l'outil, et augmente la gâche. Plus les angles sont arrondis ou la forme courbe, plus le contact est progressif et la vitesse peut être élevée.

Disposition des poses : aligné vs quinconce

En complément du rayon, le moteur peut décaler les poses en quinconce sur la bobine pour répartir les points d'impact dans le temps :

  • Pose alignée (par défaut) → tous les bords d'étiquettes frappent le squelette simultanément à chaque tour → coef quinconce = 1,00
  • Pose en quinconce (décalée 1 ligne sur 2) → impacts répartis → coef quinconce = 1,10
— Argument commercial fort

Le quinconce est invisible pour le client final : chaque étiquette individuelle reste strictement identique. C'est uniquement la disposition de production sur la bobine. Donc aucune objection client possible — c'est purement une optimisation technique côté imprimeur. Bonus vitesse +10 % "gratuit" si le format le permet.

Intégration dans l'algorithme

Le coefficient confort de roulage s'applique multiplicativement au coefficient de vitesse final, à côté du scoring échenillage et de la consolidation. Tous les bonus/pénalités se cumulent.

// Calcul du coef vitesse final, combinant toutes les règles
coef_vitesse_final = coef_vitesse_echenillage
                   × coef_consolidation
                   × coef_confort_roulage_rayon
                   × coef_quinconce

// Exemple : étiquette rectangle 100×100 rayon 3 mm, pose alignée,
//           intervalle dev 4 mm (qualité bien)
//   coef_echenillage = 1,00
//   coef_consolidation = 1,00 (pas nécessaire)
//   coef_rayon = 1,02
//   coef_quinconce = 1,00
//   → coef final = 1,02 (vitesse +2 % vs nominale)

// Exemple : même étiquette en rayon 5 mm + quinconce
//   coef final = 1,00 × 1,00 × 1,08 × 1,10 = 1,188 (+19 %)
//   → significativement plus rapide, économie sur le temps machine

Propositions automatiques au commercial

Si une étiquette à angles vifs ou rayon 1-2 mm est demandée, le moteur peut proposer :

  • "Augmenter le rayon à 5 mm" — économie X € sur cette commande (souvent acceptable visuellement par le client, à vérifier sur BAT)
  • "Activer la pose en quinconce" — économie Y € sans aucun impact visuel client (décision purement interne imprimeur)

Ces propositions apparaissent dans la section "Optimisations possibles" de l'écran d'optimisation de pose, à côté des propositions de format alternatif. Le commercial coche ce qu'il accepte de proposer au client (rayon) ou ce qu'il active directement (quinconce, ne nécessite pas l'accord client).

🎯 Contrainte client final — intervalle de détection machine de pose

— Le minimum est imposé par les DEUX côtés

L'intervalle développé optimal côté imprimeur (production flexo) est faible : 2-3 mm suffisent. Mais le client final qui pose les étiquettes a une contrainte différente : sa machine d'étiquetage automatique utilise une cellule photoélectrique qui doit détecter chaque étiquette individuellement pour la poser au bon endroit sur son produit. Si l'écart est trop faible, la cellule confond deux étiquettes consécutives → pose ratée.

L'intervalle dev réel appliqué = MAX(min imprimeur, min client). Si le client a besoin de 4 mm, on applique 4 mm même si techniquement on pourrait faire 2 mm en production.

👤 Principe transversal — souveraineté commerciale

— Le moteur propose, l'humain décide

À aucun moment FlexoCompare n'impose une valeur calculée. Le moteur d'optimisation, la table échenillage, le barème de compensation, le filtre effet banane sont des recommandations. Le commercial conserve la souveraineté complète sur :

  • Le choix final de la configuration (peut prendre la #3 plutôt que la #1)
  • Les intervalles dev et laize (peut saisir des valeurs hors recommandation)
  • Les coefficients de vitesse et de gâche (peut les ajuster manuellement pour ce devis)
  • Le forçage d'un cylindre exclu par effet banane (si l'imprimeur sait que sa machine tolère)
  • L'ignorance d'une alerte qualité (production dégradée acceptée pour ce client)
  • L'ajout d'un surcoût manuel pour compenser une exigence client

— Workflow subi

🔪 Postes de découpe — découpe recto + verso mi-chair

— Concept clé

Les machines flexo modernes (Mark Andy P9, OMET XFlex, Nilpeter FA-22, Gallus Labelmaster…) embarquent souvent 2 stations de découpe rotative inline. Ça permet de réaliser en une passe :

  • Découpe recto mi-chair (kiss-cut classique) — découpe l'étiquette mais pas le liner. Standard.
  • Découpe verso mi-chair — découpe partielle au verso pour étiquettes livret, peel-and-reveal, multicouches
  • Découpe pleine (cut through) — pour étiquettes feuilles ou paravents
  • Combinaisons — mi-chair recto + mi-chair verso = livret en une passe

Conséquences sur le moteur :

  • La machine a un champ nb_postes_decoupe (1 ou 2)
  • Le devis a un champ nb_outils_decoupe_requis (1 ou 2)
  • Si nb_outils_requis > nb_postes_machine → la machine est filtrée du moteur d'optimisation (pas compatible)

Calcul cumulatif des impacts dans le moteur

Tous les coefficients (règles métier + options de fabrication) se cumulent multiplicativement pour donner la vitesse et la gâche finales :

// Coefficient vitesse final, intégrant TOUTES les sources d'impact
coef_vitesse_final = coef_echenillage 
                   × coef_consolidation 
                   × coef_confort_rayon 
                   × coef_quinconce 
                   × PRODUIT(option.coef_vitesse_impact 
                             pour chaque option DANS devis.options_selectionnees)

// Coefficient gâche final
coef_gache_final = coef_gache_echenillage 
                 × PRODUIT(option.coef_gache_impact 
                           pour chaque option DANS devis.options_selectionnees)

// Temps de calage total
temps_calage_total = temps_calage_base
                   + nb_couleurs × temps_calage_par_couleur
                   + SOMME(option.ajoute_temps_calage_min 
                           pour chaque option DANS devis.options_selectionnees)

// Application au calcul final
vitesse_reelle = machine.vitesse_pratique_m_min × coef_vitesse_final
gache_reelle_pct = gache_base × coef_gache_final
temps_production_h = (qte_metres / vitesse_reelle) + (temps_calage_total / 60)
cout_machine_total = temps_production_h × machine.cout_horaire_eur

Exemple de cumul (étiquette complexe)

— Étiquette livret + dorure chaud + spot détection verso, sur machine 80 m/min
  • Coef échenillage (intervalle 3 mm) = 1,00
  • Coef consolidation = 1,00
  • Coef confort rayon (rayon 3 mm) = 1,02
  • Coef quinconce = 1,00
  • Coef option livret = 0,70
  • Coef option dorure chaud = 0,75
  • Coef option spot verso = 0,95

Coef vitesse final = 1,00 × 1,00 × 1,02 × 1,00 × 0,70 × 0,75 × 0,95 = 0,51

→ Vitesse réelle : 80 × 0,51 = ~41 m/min (au lieu des 80 nominaux).
Temps de calage cumulé : base 20 min + livret 30 min + dorure 25 min + spot 5 min = ~80 min de calage avant production.
Coût total significativement majoré, ce qui se reflète dans le devis final.

📖 Focus technique — étiquettes livret (booklet)

— Construction d'une étiquette 2 couches qui s'ouvre

Contrairement à ce qu'on pourrait croire, le livret ne nécessite pas 2 outils de découpe. Il s'obtient par un processus de retournement de laize + lamination contrôlée avec vernis spécial libérateur. Le processus :

  1. Impression standard de la laize complète avec les étiquettes (1 seul outil de découpe pour la forme)
  2. Vernis spécial #1 (libérateur / pelable) appliqué sur la moitié inférieure de la laize
  3. Module retournement : la bande est coupée en deux dans le sens longitudinal et l'une des moitiés est retournée sur l'autre
  4. Vernis spécial #2 appliqué sur la moitié supérieure pour la lamination contrôlée
  5. Relamination : les deux moitiés se collent via les vernis, mais peuvent être séparées et refermées par le client final

Résultat : une étiquette 2 couches qui s'ouvre comme une page. Cas d'usage typiques : étiquettes notice multilingue (cosmétique, pharma), instructions techniques détaillées, étiquettes promotionnelles "ouvrez pour découvrir".

Pré-requis machine pour le livret

  • 2 stations vernis minimum (pas vernis CMJN classique, vernis spécial libérateur — souvent acrylique pelable)
  • Module retournement de laize intégré inline (équipement spécifique, présent sur Mark Andy P9, OMET XFlex, certains Gallus Labelmaster — pas sur toutes les machines)
  • 1 seul outil de découpe suffit (forme standard de l'étiquette finale)
  • Les vernis spéciaux sont des consommables dédiés à acheter chez fournisseurs spécialisés (Actega, Sun Chemical, INX) — à intégrer au catalogue encres/vernis

🔄 Focus technique — découpe split-liner

— Technique de découpe verso sous support

Contrairement à la découpe recto mi-chair (qui découpe l'étiquette en laissant le liner intact), la découpe split-liner fait l'inverse : un outil positionné en VERSO sous le support découpe uniquement le liner, sans toucher à l'étiquette imprimée au-dessus. Ce qui demande une précision extrême et une synchronisation parfaite entre les deux postes de découpe.

Variantes courantes :

  • Split-back centré : 1 ligne de coupe au milieu du liner → le client retire une moitié, positionne l'étiquette, retire l'autre moitié. Utilisé pour grandes étiquettes cosmétique, bouteilles >60 mm.
  • Perforation liner : ligne perforée (pas une coupe complète) → permet la séparation manuelle facile, étiquettes livret ou paravents.
  • Coupes multiples : plusieurs lignes parallèles → applications accordéon, étiquettes industrielles à plusieurs sections détachables.

Contraintes techniques split-liner

  • Machine : 2 postes de découpe minimum obligatoire (1 recto + 1 verso). Filtre dur dans le moteur.
  • Précision : le filet de l'outil verso doit avoir une hauteur calibrée pour découper UNIQUEMENT le liner (épaisseur typique 65 µ) sans entamer la couche adhésive ni l'étiquette.
  • Silhouette auto : la position de coupe est spécifique au format et au besoin client → est_silhouette = TRUE pour la plaque verso.
  • Plaque dédiée : 2 plaques sont à gérer (recto + verso), liées entre elles. Le catalogue plaques doit pouvoir représenter cette liaison.
  • Calage : le BAT machine doit valider l'alignement parfait des 2 découpes avant production.

Impact sur le modèle de données

Les plaques recto et verso liées sont représentées par une relation parent-enfant dans plaques_decoupe :

ALTER TABLE plaques_decoupe ADD COLUMN position_decoupe VARCHAR(10) DEFAULT 'recto', -- recto, verso ADD COLUMN plaque_liee_id UUID REFERENCES plaques_decoupe(id), -- ex: plaque recto liée à plaque verso ADD COLUMN type_liaison VARCHAR(20); -- split_liner, livret, etc.

Quand le commercial sélectionne l'option "Découpe split-liner" sur un devis, le moteur sait qu'il faut chercher / commander DEUX plaques : la plaque recto pour la découpe mi-chair, et la plaque verso pour la coupe liner. Les deux sont mémorisées comme liées pour les commandes récurrentes.

— 03d / Rebobinage & finition bobine

Rebobinage : de la bobine mère aux bobines clients

Après production, la bobine mère sortie de presse contient plusieurs pistes (poses laize) côte à côte sur une grande largeur. Le rebobinage est l'étape qui sépare ces pistes en bobines clients distinctes, conformes aux contraintes de leur machine de pose : diamètre mandrin, diamètre maxi, sens d'enroulement, nombre d'étiquettes par bobine, marquage.

— Étape obligatoire et facturable

Le rebobinage est une étape de production supplémentaire avec sa propre machine (rebobineuse / slitter rewinder), son temps machine, son coût horaire et sa gâche en raccord. Le moteur de devis intègre automatiquement ces coûts à partir des paramètres bobine du client.

Paramètres bobine — propriété du client

Ces données appartiennent au profil client, pas au devis. Le client a SA machine de pose avec SES contraintes ; elles ne changent pas d'une commande à l'autre. Stockés dans le catalogue clients :

Les 4 sens d'enroulement

— Vocabulaire métier standardisé

Chaque sens d'enroulement décrit comment l'étiquette se présente quand la bobine se déroule sur la machine de pose :

  • SE1 : étiquette face extérieure, défilement endroit (sens de lecture normal en sortie)
  • SE2 : étiquette face extérieure, défilement inversé (sens de lecture tête en bas)
  • SE3 : étiquette face intérieure, défilement endroit
  • SE4 : étiquette face intérieure, défilement inversé

Le commercial doit demander au client son sens préféré et le stocker dans le profil client. Erreur classique en imprimerie : livrer SE1 alors que le client attendait SE2 → la machine de pose pose les étiquettes à l'envers → rejet de toute la commande.

🪵 Approvisionnement mandrins — interne ou externe

— Deux modèles industriels possibles

Chaque imprimerie choisit son modèle d'approvisionnement mandrins. Ce choix impacte directement le coût et le temps de production :

  • Mode A — Achat de mandrins pré-coupés : commande chez fournisseurs spécialisés (Sonoco, Cores France, Mondi, distributeurs locaux). Coût unitaire 0,30 à 0,80 €/mandrin selon diamètre et longueur. Zéro temps opérateur. Stockage multiple longueurs nécessaires.
  • Mode B — Achat de tubes-mères + découpe interne : tubes longs 1000-3000 mm achetés en gros, découpés à la demande. Coût matière divisé par 2 à 4 vs pré-coupé. Nécessite machine de découpe (scie à mandrins manuelle, semi-auto, ou tronçonneuse rotative).

🧠 Arbitrage dynamique par devis (système intelligent)

— Calcul de rentabilité CHAQUE devis, pas annuel

Si l'imprimerie dispose d'une scie ET pratique les deux modes, FlexoCompare arbitre commande par commande en temps réel. Pour une petite série, le pré-coupé peut être plus rentable (le temps de réglage scie n'est pas amorti). Pour une grosse série, la découpe interne devient rentable. Le moteur calcule les deux options et propose la meilleure au commercial.

Algorithme d'arbitrage par devis

FONCTION choisir_mode_mandrins(nb_bobines, longueur_mandrin_mm, diametre_mandrin_mm, 
                              parametres_mandrins, stock_mandrins_dispo) :

  // CAS 1 — Pas de scie disponible → mode forcé
  SI parametres_mandrins.scie_disponible == FALSE :
    RETOURNER {
      mode: "pre_coupe",
      raison: "Pas de scie à mandrins disponible",
      cout_total: calcul_cout_pre_coupe(nb_bobines, ...)
    }

  // CAS 2 — Stock dispo couvre la commande
  stock_dispo = stock_mandrins_dispo[diametre][longueur_mandrin_mm]
  SI stock_dispo >= nb_bobines :
    RETOURNER {
      mode: "stock",
      raison: "Mandrins déjà en stock (" + stock_dispo + " unités)",
      cout_total: nb_bobines × cout_amortissement_stock
    }

  // CAS 3 — Calcul comparatif des deux modes pour cette commande
  
  // Option A : pré-coupé
  prix_unit_pre_coupe = parametres_mandrins.tarifs_pre_coupes[diametre][longueur].prix
  cout_total_A = nb_bobines × prix_unit_pre_coupe
  temps_total_A = 0  // pas de temps opérateur
  delai_dispo_A = parametres_mandrins.delai_livraison_fournisseur_jours  // typique 3-7j
  
  // Option B : découpe interne
  cout_matiere_par_unit = (longueur_mandrin_mm / 1000) × tubes_meres[diametre].prix_metre
  temps_decoupe_min = parametres_mandrins.temps_reglage_initial_min 
                    + (nb_bobines × parametres_mandrins.temps_par_coupe_sec / 60)
  cout_main_oeuvre = (temps_decoupe_min / 60) × parametres_mandrins.cout_horaire_scie_eur
  cout_total_B = (nb_bobines × cout_matiere_par_unit) + cout_main_oeuvre
  delai_dispo_B = 0  // disponible immédiatement
  
  // Calcul du seuil de bascule pour info
  seuil_bascule = parametres_mandrins.temps_reglage_initial_min 
                × parametres_mandrins.cout_horaire_scie_eur / 60
                / (prix_unit_pre_coupe - cout_matiere_par_unit 
                   - (parametres_mandrins.temps_par_coupe_sec / 3600 
                      × parametres_mandrins.cout_horaire_scie_eur))
  
  // Décision
  SI cout_total_A <= cout_total_B :
    mode_recommande = "pre_coupe"
    economie = cout_total_B - cout_total_A
  SINON :
    mode_recommande = "decoupe_interne"
    economie = cout_total_A - cout_total_B
  
  RETOURNER {
    mode: mode_recommande,
    raison: "Optimal pour " + nb_bobines + " bobines (seuil bascule ~" + seuil_bascule + ")",
    cout_total: MIN(cout_total_A, cout_total_B),
    economie_vs_autre_mode: economie,
    comparatif: {
      pre_coupe: {cout: cout_total_A, delai: delai_dispo_A + "j"},
      decoupe_interne: {cout: cout_total_B, temps_atelier: temps_decoupe_min + "min", delai: "immédiat"}
    },
    suggestion_batch: SI mode_recommande == "decoupe_interne" ET nb_bobines < 10 :
                       "Penser à batcher avec d'autres commandes pour amortir le réglage"
                     SINON : null
  }

— 03f / Contrôle BAT automatisé par IA

Vérification du 1er tirage vs BAT au démarrage production

Au démarrage de chaque production, l'opérateur photographie le 1er tirage sortant de presse. FlexoCompare le compare automatiquement au BAT validé par le client via Claude API multimodal. En quelques secondes, le système confirme la conformité ou pointe les écarts à corriger avant de lancer la production en cadence. Économie typique : 200 à 1 000 € de gâche par démarrage problématique évité.

— Pourquoi c'est puissant

Aujourd'hui, le contrôle BAT vs 1er tirage se fait à l'œil par l'opérateur ou le chef d'atelier. C'est subjectif, ça prend du temps, et un défaut subtil peut passer inaperçu — entraînant 10 000 étiquettes défectueuses avant détection. Avec FlexoCompare, l'IA pointe systématiquement les écarts, même mineurs, et ne laisse jamais démarrer une prod douteuse.

Workflow

  1. Upload BAT de référence : à la validation client (avant production), le BAT est lié au devis dans FlexoCompare (PDF ou image scanée)
  2. Au démarrage prod : l'opérateur sort le 1er tirage de la presse et le photographie (smartphone, scanner inline, caméra atelier)
  3. Comparaison Claude API : envoi des deux images avec un prompt comparatif détaillé
  4. Retour structuré : score de conformité global + liste des écarts détectés avec localisation
  5. Décision opérateur : valide, ajuste, ou rejette → traçabilité complète

📐 Protocole de prise de photo standardisé

— Position opérateur = double contrôle

L'opérateur se place toujours face à la sortie machine, dans la position standardisée définie pour la presse. Cette contrainte n'est pas anodine : elle permet à l'IA de contrôler en plus du visuel le sens de défilement de la bande, qui détermine directement le sens d'enroulement après rebobinage (SE1/SE2/SE3/SE4).

Détecter un mauvais sens au 1er tirage évite de produire 30 000 étiquettes dans le mauvais sens, impossibles à reconditionner correctement pour le client.

Référentiel de position

  • Face à la sortie machine : opérateur perpendiculaire à la direction de défilement
  • Distance fixe : ~1 m de la sortie (pour cadrage homogène)
  • Hauteur photo : au niveau de la bande (pas en plongée ou contre-plongée)
  • Sens de défilement : la bande sort du fond vers l'opérateur (top-down dans l'image)
  • Éclairage : éclairage atelier standard, pas de flash direct (risque de reflet sur vernis/dorure)

Ce protocole est documenté dans le système avec une image de référence pour chaque machine du parc (l'opérateur peut la consulter avant la photo).

— 05 / Recherche

Moteur de recherche plaque

Le moteur de recherche est le cœur du système. Il s'exécute automatiquement dès qu'une demande de devis est finalisée à l'étape 4, et propose au commercial les plaques disponibles ou alerte sur la nécessité d'une commande externe.

Algorithme principal

FONCTION rechercher_plaque(specs_devis, parametres_entreprise) :

  // Étape 1 — Le besoin est-il "silhouette" ?
  silhouette = specs_devis.est_silhouette

  // Étape 2 — Périmètre de recherche
  SI silhouette :
    perimetre = plaques OÙ client_proprietaire_id == specs_devis.client_id
                       ET est_silhouette == TRUE
  SINON :
    perimetre = plaques OÙ est_silhouette == FALSE

  // Étape 3 — Filtres obligatoires
  perimetre = perimetre OÙ
    matiere_compatible == specs_devis.matiere
    ET statut == "actif"
    ET nb_tours_utilises < (nb_tours_garantie * 0.85)

  // Étape 4 — Recherche exacte d'abord
  matches_exact = perimetre OÙ
    format_laize_mm == specs_devis.laize
    ET format_devel_mm == specs_devis.devel
    ET intervalle_laize_mm == specs_devis.intervalle_laize
    ET intervalle_devel_mm == specs_devis.intervalle_devel
    ET rayon_angles_mm == specs_devis.rayon

  SI matches_exact NOT EMPTY :
    RETOURNER {
      type: "EXACT",
      plaques: matches_exact triés par moins_usé_d_abord,
      economie_estimee: cout_plaque_neuve
    }

  // Étape 5 — Recherche approchante (tolérance)
  tol_laize = parametres_entreprise.tolerance_laize_mm  // default 2 mm
  tol_devel = parametres_entreprise.tolerance_devel_mm  // default 2 mm
  tol_int = parametres_entreprise.tolerance_intervalle_mm  // default 0.5 mm

  matches_approchant = perimetre OÙ
    ABS(format_laize_mm - specs_devis.laize) <= tol_laize
    ET ABS(format_devel_mm - specs_devis.devel) <= tol_devel
    ET ABS(intervalle_laize_mm - specs_devis.intervalle_laize) <= tol_int
    ET ABS(intervalle_devel_mm - specs_devis.intervalle_devel) <= tol_int
    ET rayon_angles_mm == specs_devis.rayon

  SI matches_approchant NOT EMPTY :
    RETOURNER {
      type: "APPROCHANT",
      plaques: matches_approchant triés par proximité_format,
      economie_estimee: cout_plaque_neuve,
      requiert_validation_client: TRUE,
      ecart_max_mm: max écart format laize+devel
    }

  // Étape 6 — Aucune correspondance
  RETOURNER {
    type: "AUCUNE",
    action: "commander_plaque_neuve",
    cout_estime: parametres_entreprise.cout_plaque_neuve_default,
    delai_estime_jours: parametres_entreprise.delai_fabricant_default

— 06 / Usure

Compteur d'usure plaque

Garantie standard fournisseur

Les fabricants de plaques (LARTEC, Drupa, Channel, etc.) garantissent typiquement 1 000 000 de tours de rotation. Au-delà, la plaque commence à se déformer, les coupes deviennent imprécises, et il faut renouveler.

Décompte automatique

Pour chaque commande utilisant une plaque P :
  tours_consommes_commande = quantité_étiquettes_produites
                           / (pose_laize × pose_devel)
  P.nb_tours_utilises += tours_consommes_commande
  
  SI P.nb_tours_utilises >= P.nb_tours_garantie * 0.85 :
    P.statut = "use_85pct"
    Envoyer alerte au responsable atelier
  
  SI P.nb_tours_utilises >= P.nb_tours_garantie :
    P.statut = "fin_de_vie"
    P n'est plus proposée par le moteur de recherche

Amortissement par devis (calcul de marge fine)

Quand une plaque est réutilisée pour un devis, on peut imputer un coût d'amortissement proportionnel au nombre de tours consommés :

amortissement_plaque = (P.prix_achat / P.nb_tours_garantie)
                     × tours_consommes_commande

Ce coût est ajouté automatiquement au devis de manière transparente pour le commercial. Sur 1 000 000 tours à 800 € de plaque, ça fait 0,0008 €/tour, mais sur des commandes de 500 000 tours c'est 400 € de marge consommée qu'il ne faut pas oublier de récupérer.

Paramètres configurables (par imprimerie)

  • Nombre de tours garantis par défaut (default 1 000 000)
  • Seuil d'alerte usure en % (default 85 %)
  • Coefficient d'amortissement : linéaire (default) ou progressif
  • Coût d'amortissement répercuté ou non au client (toggle)
— 07 / Matières

Catalogue matières

Pourquoi un catalogue dédié

La matière du substrat est le critère qui détermine si une plaque est techniquement compatible. Sans catalogue centralisé, on aurait des chaînes de caractères libres ("Couché 80g", "couche 80", "papier couché blanc 80g") qui rendraient impossibles les comparaisons.

Le catalogue matière fait référence : chaque plaque a un matiere_compatible_id qui pointe vers une entrée du catalogue. Chaque demande de devis sélectionne une matière dans ce même catalogue. La comparaison devient triviale.

Structure d'une matière

  • ID, code court, libellé long
  • Catégorie : papier / film plastique / synthétique / spécial
  • Sous-type : couché, vélin, thermique, BOPP, PET, PVC, vinyle, etc.
  • Grammage (g/m²) ou épaisseur (µ)
  • Adhésifs compatibles (liste)
  • Certifications sanitaires (FDA, EU 10/2011, BPF, etc.)
  • Certifications environnementales : FSC, PEFC, EcoLabel européen, Cradle to Cradle
  • Fournisseurs habituels (liste avec prix au m²)
  • Notes techniques (filet adapté, particularités)

🔍 Spot de détection au verso — matières transparentes

— Cas spécifique des films transparents

Les cellules photoélectriques standards ne détectent pas les films transparents (PET clair, BOPP transparent, polyéthylène clair) : la lumière passe à travers la matière sans signal. Pour permettre la pose automatique chez le client, on imprime un spot opaque au verso de chaque étiquette (souvent noir mat ou aluminium, 5×5 mm typiquement), qui sert de repère à la cellule.

Le spot est invisible côté visuel client (au dos), mais représente une couleur d'impression supplémentaire dans la production : +1 cliché, +1 groupe machine à caler, +1 encre. Le surcoût est intégré au devis automatiquement.

Règle automatique

SI matiere.est_transparent == TRUE
ET client.cellule_detection NOT IN ("optique_haute_resolution_transparent_capable") :
   → AUTO-RECOMMANDER spot_detection_verso
   → AJOUTER +1 couleur dans le calcul devis
   → AJOUTER +1 cliché dans commande clichés (cas A et B du workflow externe)
   → MENTIONNER "Spot détection 5×5 mm noir au verso" sur fiche technique atelier

Champs additionnels matière

Sur chaque entrée du catalogue matières, deux champs viennent s'ajouter pour gérer la transparence :

  • est_transparent (boolean) — si TRUE, le moteur déclenche la règle automatique
  • opacite_pct (numeric optionnel) — % d'opacité de la matière (100 % = totalement opaque, 0 % = totalement transparent), pour affiner la décision sur les matières semi-transparentes

Champs additionnels client (catalogue clients)

Sur chaque entrée du catalogue clients, en complément du type de cellule de détection :

  • cellule_detecte_transparent (boolean) — si TRUE, le spot n'est pas requis
  • position_spot_preferee (varchar) — position standardisée par client : centre, coin haut-gauche, etc. (chaque client a sa convention selon sa machine)
  • couleur_spot_preferee (varchar) — noir mat / aluminium / autre
  • dimensions_spot_mm (varchar) — typiquement "5x5" mais peut varier selon la cellule

Certifications environnementales — détail

Sur chaque matière du catalogue, on enregistre quelles certifications environnementales elle porte intrinsèquement (selon son fournisseur). Exemples :

  • "Papier couché blanc 80g — Fournisseur Arconvert" → FSC mixte + PEFC
  • "Papier vélin 90g — Fournisseur UPM" → FSC uniquement
  • "BOPP transparent 50µ — Fournisseur Innovia" → aucune (films plastiques rarement certifiés bois)

Et côté paramètres entreprise, l'imprimerie déclare ses propres certifications de chaîne de contrôle, avec n° de certificat, organisme certificateur (Bureau Veritas, SGS, FCBA…), et date d'expiration. Cette information est vérifiée par le moteur de devis et apparaît sur le PDF de devis si le client a demandé une certification.

Au lancement V1, on fournit un catalogue matière de base (50–100 matières standard du marché) que chaque imprimerie peut amender / supprimer / enrichir selon ses propres références fournisseurs.

— Note importante sur la vitesse

Le champ vitesse_pratique_m_min est obligatoirement saisi manuellement par l'imprimerie. Il ne s'agit pas de la vitesse maxi constructeur (donnée catalogue, peu réaliste en production réelle) mais de la vitesse à laquelle l'imprimerie tourne effectivement sa machine en pratique. Selon les ateliers, la même Mark Andy 2200 peut tourner à 30, 60 ou 80 m/min selon le conservatisme de l'opérateur, l'état de la machine, le type de matière dominante et la qualité visée. C'est ce paramètre qui détermine le temps de production réel et donc le coût machine du devis.

Cas A — Client avec fichier vectorisé

1
Réception du fichier client (AI, PDF HD, EPS) — upload dans FlexoCompare
2
Création d'une commande externe au PHOTOGRAVEUR avec le fichier en pièce jointe
3
Génération automatique d'un PDF de commande (modèle ICE actuel adaptable par imprimeur)
4
Photograveur retourne : tracé de découpe + fichiers clichés par couleur
5
Validation BAT par client (V3 = signature électronique, V2 = email manuel)
6
Création d'une commande externe au FABRICANT DE PLAQUES avec tracé en PJ
7
Réception physique de la plaque → enregistrement automatique dans le catalogue avec n° attribué
8
Lancement production étiquettes → décompte tours plaque

Cas B — Format simple

1
Pas de fichier client particulier (format standard rectangulaire)
2
Création d'une commande externe au FABRICANT DE PLAQUES directement avec les specs (format, matière, intervalle, rayon, dents)
3
Réception physique → enregistrement catalogue
4
Lancement production

Statuts d'une commande externe

  • brouillon → en cours de saisie
  • envoyee → bon de commande PDF envoyé au fournisseur
  • accusee_reception → fournisseur a confirmé
  • en_production → en cours de fabrication chez le fournisseur
  • livree → reçue physiquement chez l'imprimeur
  • integree → enregistrée dans le catalogue, lien créé avec la plaque
  • annulee → annulée avant production
  • litige → problème qualité ou livraison

— 14 / Validation

Questions ouvertes pour l'imprimerie pilote

Ce document est un point de départ basé sur le savoir-faire ICE d'Eric. L'imprimerie pilote (ton pote, ou le premier early adopter) doit valider, amender, compléter selon SES particularités. Voici les questions explicites à lui poser.

Sur les plaques & matières

  1. Combien de plaques dans votre catalogue actuel ? Sous quel format (Excel, papier, ERP) ?
  2. Avez-vous des règles "silhouette" différentes de celles d'ICE ? (rayons standards autres que 2 et 3, par exemple)
  3. Quelle est votre garantie standard fournisseur en tours ? (1 M est la norme)
  4. Répercutez-vous l'amortissement de la plaque dans le prix client, ou est-ce déjà inclus dans votre marge brute ?
  5. Avez-vous des matières "exotiques" que le catalogue de base ne couvrira pas ?
  6. Êtes-vous certifié FSC, PEFC, Imprim'Vert, ISO 14001, ou autres ? Quels n° de certificat, organismes, dates d'expiration ? Combien de % de vos commandes concernent des étiquettes avec certification environnementale demandée par le client ?

Sur le moteur de recherche

  1. Quelle tolérance de "format approchant" appliquez-vous aujourd'hui à la main ? (chez ICE c'était ~2 mm)
  2. Acceptez-vous toujours de proposer un format approchant au client, ou seulement sur des commandes > certain seuil ?
  3. Comment gérez-vous aujourd'hui les plaques "silhouette" appartenant à un client : facturation à part, stockage spécial, durée de conservation ?

Sur le parc machine & le moteur d'optimisation

  1. Quelle est la liste exacte de vos cylindres magnétiques (développés en mm) ? Combien de porte-clichés de chaque format ?
  2. Combien de machines en production ? Marque, modèle, laize utile, vitesse maxi, nombre de groupes couleurs, options (UV / dorure / découpe rotative…) ?
  3. Quel est votre coût horaire machine par machine (paramètre commercial sensible mais essentiel pour le devis) ?
  4. Faites-vous déjà manuellement des propositions de "format alternatif" à vos clients pour optimiser la production ? Si oui, à partir de quel écart vous estimez la suggestion pertinente ?
  5. Combien de cylindres avez-vous achetés / changés ces 3 dernières années ? Sur quels critères (commande client / amortissement / opportunité) ?
  6. Pensez-vous qu'une alerte automatique "achetez un cylindre X mm, ROI 6 mois" vous serait utile, ou préférez-vous garder ce niveau de décision en humain ?

Sur le stockage

  1. Comment êtes-vous organisé physiquement ? Armoires nommées, rayonnage, casiers numérotés ?
  2. Faites-vous un inventaire annuel, et combien de temps cela vous prend ?
  3. Avez-vous déjà perdu des plaques dans le passé (en stock mais introuvables) ?
  4. Conservez-vous les plaques mortes ou les jetez-vous ? (problème de recyclage métal)

Sur le workflow commande externe

  1. Avec quels photograveurs travaillez-vous ?
  2. Avec quels fabricants de plaques ?
  3. Quel est leur délai moyen ?
  4. Avez-vous déjà des modèles de bon de commande à respecter (en-tête, mentions légales, etc.) ?

Sur l'usage souhaité de FlexoCompare

  1. Combien d'utilisateurs simultanés ? (commercial, atelier, direction, prépresse)
  2. Quels rôles & permissions ? (qui peut voir les prix d'achat, qui peut modifier les plaques, qui peut générer un devis…)
  3. Interconnexion avec votre ERP actuel souhaitée ? (CERM, Cégid, autre)
  4. Tarif acceptable par mois sachant les services rendus ? (notre cible 99–199 €/mois)
