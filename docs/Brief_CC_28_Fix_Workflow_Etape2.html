4. Fix #2 — Suppression filtre effet banane

— Décision métier

Le moteur explore TOUTES les combinaisons physiquement faisables (laize plaque ≤ laize utile machine, dev ≥ Δ min). Aucun blocage ni pénalité de score liés à l'effet banane. Souveraineté commerciale : l'utilisateur voit tout, il décide.

4.1 Backend — retirer le filtre/pénalité

À identifier dans le code (probablement dans backend/app/services/optimisation_pose.py ou cost_engine.py) :

  • Toute condition du type if nb_dents < SEUIL: skip / penalize
  • Tout calcul z_mini_effet_banane qui bloque l'ajout d'un candidat
  • Toute pénalité de score appliquée pour petit cylindre

Procédure :

  1. Grep banane, z_mini, effet dans backend/app/services/
  2. Lister les occurrences trouvées dans le commit message du fix #2
  3. Retirer la logique bloquante (filtre + pénalité de score)
  4. Conserver uniquement les filtres physiques durs (laize plaque ≤ laize utile machine, intervalle dev min, etc.)
⚠️ Sacred — cost_engine

Les valeurs EXACT V1a (1 449,09 € + 6,92 €/1000), V1b (1 921,09 €), V1b silhouette (2 109,89 €), V7a (6,85 €/1000), V8a-e doivent rester intactes. Si le filtre banane est dans cost_engine, vérifier que sa suppression ne change AUCUNE des valeurs EXACT par lot. Si doute → STOP et validation responsable produit.
