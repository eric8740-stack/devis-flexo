1. Contexte métier

En flexo, il est très fréquent qu'une commande client soit fractionnée en plusieurs lots de production avec configurations différentes :

  • Urgence client : on lance ce qu'on a en stock immédiatement, on finit avec matière qui arrive en J+2
  • Optimisation matière : un lot court avec laize réduite + un lot principal en laize standard
  • Dépannage : matière A pour les 10 000 premières, matière B pour les 5 000 suivantes

Le moteur actuel mono-config ne couvre pas ce cas. Cette refonte permet à un devis de référencer N lots de production indépendants.

Logique nb_poses_laize_force backend

  • Si nb_poses_laize_force = null → comportement actuel (le moteur teste [max, max-1, max-2])
  • Si nb_poses_laize_force = N → le moteur bloque sur N poses laize et calcule l'intervalle laize résultant
  • Validation : N >= 1 et N * format.largeur <= mach.laize_utile sinon candidat skippé avec log
  • Si intervalle laize résultant hors plage normale (< 2 mm ou > 10 mm) → alerte non-bloquante affichée sur la ligne

6. Agrégateur cost_engine multi-lots

⚠️ Sacred invariant — logique cost_engine inchangée

L'agrégateur appelle cost_engine.calculer() N fois (une fois par lot, paramètres lot indépendants). Les valeurs EXACT V1a (1 449,09 €), V1b (1 921,09 €), V1b silhouette (2 109,89 €), V7a (6,85 €/1000), V8a-e sont préservées par lot. Aucune modification de la logique cost_engine. Le moteur reste un service stateless qui calcule un coût pour une config donnée.

# backend/app/services/cost_engine_aggregator.py  ← NEW

from app.services import cost_engine
from app.models.lot_production import LotProduction
from dataclasses import dataclass
from typing import List

@dataclass
class CoutLot:
    lot_id: int
    cout_ht: float
    details: dict  # détail brut cost_engine pour audit

@dataclass
class CoutAgrege:
    cout_total_ht: float
    details_par_lot: List[CoutLot]

def calculer_devis_multilots(lots: List[LotProduction]) -> CoutAgrege:
    """Agrège les coûts de N lots de production.

    NE modifie PAS la logique cost_engine. Appelle simplement le moteur
    pour chaque lot avec ses propres paramètres (cylindre, machine,
    poses, quantité, matière).
    """
    couts_lots = []
    for lot in lots:
        detail = cost_engine.calculer(
            cylindre=lot.cylindre,
            machine=lot.machine,
            nb_poses_dev=lot.nb_poses_dev,
            nb_poses_laize=lot.nb_poses_laize,
            quantite=lot.quantite,
            matiere=lot.matiere,
            # ... autres params nécessaires depuis le devis parent
        )
        couts_lots.append(CoutLot(
            lot_id=lot.id,
            cout_ht=detail.cout_total_ht,
            details=detail.to_dict(),
        ))

    cout_total = sum(c.cout_ht for c in couts_lots)
    return CoutAgrege(cout_total_ht=cout_total, details_par_lot=couts_lots)

    @validator('lots')
    def somme_quantites_egale_total(cls, v, values):
        total = sum(lot.quantite for lot in v)
