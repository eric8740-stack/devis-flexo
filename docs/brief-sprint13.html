1. Contexte stratégique

Le projet pivote d'un produit unique (Devis-Flexo) vers un concept dual :

  • FlexoCompare : outil de devis intelligent (cœur historique du projet)
  • FlexoCheck : IA qualité standalone (modules d'analyse photo + contrôle BAT, vendables séparément)
  • FlexoSuite : bundle des deux à tarif préférentiel

Techniquement, c'est une seule application avec activation modulaire par utilisateur. L'utilisateur ne voit que les modules de son abonnement.

Modèle modules activés

# app/models/user.py — extension du modèle User existant

class User(Base):
    # ... champs existants
    
    # NOUVEAU : modules activés
    has_flexocompare = Column(Boolean, default=False, nullable=False)
    has_flexocheck = Column(Boolean, default=False, nullable=False)
    
    @property
    def has_bundle(self):
        return self.has_flexocompare and self.has_flexocheck
    
    @property
    def active_modules(self):
        modules = []
        if self.has_flexocompare: modules.append("flexocompare")
        if self.has_flexocheck: modules.append("flexocheck")
        return modules

Middleware d'accès modulaire

# app/api/dependencies.py — décorateur de protection de route

from fastapi import Depends, HTTPException
from app.models.user import User

def require_module(module_name: str):
    """Décorateur qui vérifie que l'utilisateur a accès au module."""
    def dependency(current_user: User = Depends(get_current_user)):
        if not getattr(current_user, f"has_{module_name}", False):
            raise HTTPException(403, f"Module {module_name} non activé")
        return current_user
    return dependency

# Usage dans les routes :
@router.post("/optimisation/calculer")
async def calculer(
    devis: DevisIn,
    user: User = Depends(require_module("flexocompare"))
):
    ...

@router.post("/ia/analyser-photo")
async def analyser_photo(
    file: UploadFile,
    user: User = Depends(require_module("flexocheck"))  # ou flexocompare selon strat
):
    ...
